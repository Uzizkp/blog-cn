# t. The following scenario is given. We want to invert 1000 2 Ã— 2 matrices.
# The 2 Ã— 2 matrices are represented by a row (see Figure 1).
# The numbers written as subscripts represent the position of the original matrix entry of the 2 Ã— 2 matrices.
# The superscript denotes the current array n âˆˆ [1 . . . 1000].
# Now, create an array with a size of 1000Ã—4 using the command rand.
# Every row in that matrix represents a 2 Ã— 2 matrix (compare Figure 1).
# Note, that the memory layout of the 2 Ã— 2 matrices must not be changed.
# That means we donâ€™t want to change the current structure of the 2 Ã— 2 matrices represented as a row.
# Donâ€™t change the 1 Ã— 4 matrices to 2Ã—2 matrices and donâ€™t change the inv command.
# We can now use Cramerâ€™s Rule to compute the inverse of our 1000, as row represented, 2 Ã— 2 matrices.
# Compute the inverses of the created 2 Ã— 2 matrices without using any loops.
T = np.matlib.rand(1000, 4)
T_inv = np.matlib.empty((1000, 4))
T_star = np.matlib.empty((1000, 4))
i = 0
while i <= 999:
	det = T[i, 0]*T[i, 3]-T[i, 1]*T[i, 2]
	T_star[i, :] = [T[i, 3], -T[i, 1], -T[i, 2], T[i, 0]]
	T_inv[i, :] = T_star[i, :] / det
	i = i+1
print('t. invert 1000 2*2 matrices :\n', T_inv)

		print('input error')
